[原创] (向量化异常处理)VEH hook

这个项目的目的是创建两个对话框窗口，一个是主对话框，有两个按钮，一个按钮负责调转至另一个对话框进行拦截（Hook）前的修改操作，另一个按钮负责调用相应的API触发Hook操作

另一个对话框再DLL程序中创建，存在三个编辑框和一个按钮，上面两个编辑框分别是输入dll名称和api名称的编辑框，旁边的按钮进行Hook的准备工作，下面的编辑框负责记录Hook操作

利用SEH结构化异常处理HOOK的大概原理是自定义一个顶层异常处理，当发生一个异常时，优先跳转至我们写好的函数中处理，那么就可以通过在相应的api函数地址开始处写入一个 INT 3 指令，下入人为断点后调用api函数就会触发异常，进入我们写好的函数中从而完成hook操作

结构化异常处理编写程序的流程：首先调用GetProcAddress获取AddVectoredExceptionHanlder函数的地址，之后在调用这个函数安装顶层异常，当触发异常后会跳转到自己编写的错误处理异常函数中，如果是手动下的int3异常，那么执行完异常处理函数后会跳转回被hook的函数继续执行。
